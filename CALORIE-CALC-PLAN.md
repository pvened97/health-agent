# Расчёт дневной нормы калорий — план итераций

Фича: автоматический расчёт дневной нормы калорий и макросов для набора мышечной массы.

Формула: BMR (Mifflin–St Jeor) × коэффициент активности + надбавка по Strain + профицит × Recovery.

## Итерация 2 — Инструмент записи веса

### Цель
Агент может записывать и читать вес пользователя. Модель `BodyMetric` в БД уже существует.

### Новые tools

**save_body_metric(weight_kg, body_fat_pct?, comment?)**
- Записывает в таблицу `body_metrics`
- Дата = сегодня (MSK)
- Если за сегодня уже есть запись — обновить, а не дублировать
- Возвращает подтверждение: "Вес 75.5 кг записан (28.03.2026)"

**get_weight_history(days=30)**
- Возвращает все записи веса за период
- Скользящая средняя за 7 дней (пригодится для калибровки в будущем)
- Формат: дата, вес, дельта с прошлым измерением

### Изменения в промпте
- Если пользователь пишет "вешу 76", "утренний вес 75.5", "взвесился — 74.8" → записывать через save_body_metric
- Обновлять `anthropometry/weight_kg` в профиле при значительном изменении (±2 кг от текущего)

## Итерация 4 — Коэффициент активности в профиле

### Цель
Агент знает базовый уровень бытовой активности пользователя (без учёта тренировок).

### Поле профиля
`lifestyle/activity_level` — добавить в `REQUIRED_PROFILE_FIELDS` после итерации 1.

### Значения и маппинг
- `low` (1.35) — офис, мало шагов, сидячий образ жизни
- `moderate` (1.45) — офис + прогулки, 6-8k шагов
- `high` (1.55) — активная работа на ногах, 10k+ шагов
- `very_high` (1.60) — физический труд, постоянное движение

### Логика определения
Агент определяет уровень из свободного текста:
- "я программист, работаю из дома" → low
- "хожу в офис, гуляю с собакой" → moderate
- "работаю курьером" → high
- "работаю на стройке" → very_high

Пользователь может в любой момент скорректировать: "у меня сейчас более активный образ жизни" → агент обновляет.

### Файлы
- `app/agent/context.py` (REQUIRED_PROFILE_FIELDS)
- `app/agent/agent.py` (промпт: правила определения уровня)

## Итерация 5 — Калькулятор дневной нормы

### Цель
Tool, который считает дневную цель по калориям и макросам.

### Формула

```
Цель_ккал = Базовое_поддержание + Поправка_на_нагрузку + Профицит_на_набор

Где:
  Базовое_поддержание = BMR × Коэффициент_активности

  BMR (муж) = 10 × вес + 6.25 × рост − 5 × возраст + 5
  BMR (жен) = 10 × вес + 6.25 × рост − 5 × возраст − 161

  Поправка_на_нагрузку = Надбавка_по_Strain × Модификатор_типа_нагрузки

  Надбавка по Day Strain:
    0–9   → 0 ккал
    10–13 → +120 ккал
    14–17 → +220 ккал
    18–21 → +320 ккал

  Модификатор типа нагрузки:
    силовая (strength training, weightlifting, functional fitness) → 1.0
    смешанная (crossfit, hiit, boxing, martial arts) → 0.85
    кардио (running, cycling, swimming, rowing) → 0.75
    лёгкая (yoga, stretching, pilates) → 0.40
    нет тренировки (strain от бытовой активности/стресса) → 0.25

  Если тренировок несколько за день → берём максимальный модификатор.

  Профицит_на_набор = Базовое_поддержание × Процент_профицита × Модификатор_Recovery

  Процент профицита: 0.05 (дефолт), настраиваемо 0.05–0.10

  Модификатор Recovery:
    Green (≥67%) → 1.0
    Yellow (34–66%) → 0.85
    Red (<34%) → 0.70
```

### Макросы (от итоговой Цель_ккал)

```
Белок = 2.0 г × вес_кг
Жиры = Цель_ккал × 0.25 / 9
Углеводы = (Цель_ккал − Белок × 4 − Жиры × 9) / 4
Клетчатка = 25–30 г (фикс, ближе к 30 при калорийности >2500)
```

### Tool: calculate_daily_target(target_date?)

Входы (автоматически из БД):
- Профиль: вес, рост, возраст, пол, activity_level, primary_goal
- CycleLog: day_strain за target_date
- WorkoutLog: тренировки за target_date (для типа нагрузки)
- RecoveryLog: recovery_score за target_date

Выход (строка для агента):
```
Дневная цель (28.03.2026):
  BMR: 1850 ккал
  Базовое поддержание (×1.45): 2683 ккал
  Надбавка по нагрузке: +220 × 1.0 (силовая) = +220 ккал
  Профицит на набор: 2683 × 5% × 1.0 (🟢 recovery 72%) = +134 ккал

  ИТОГО: 3037 ккал
  Белок: 150 г (600 ккал)
  Жиры: 84 г (759 ккал)
  Углеводы: 420 г (1678 ккал)
  Клетчатка: 30 г
```

### Fallback (нет данных WHOOP)
- Нет day_strain → поправка на нагрузку = 0, агент предупреждает
- Нет recovery → модификатор = 1.0 (считаем green по умолчанию)
- Нет тренировок → модификатор типа = 0.25

### Ограничения модели (зашить в код)
1. Strain не конвертируется в калории — только зональные надбавки
2. Recovery влияет ТОЛЬКО на профицит, не на базу
3. WHOOP calories не используются как источник истины
4. Профицит применяется только при goal = "набор массы"
